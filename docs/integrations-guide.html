Integrations Guide

Stay informed and in control. Oh My Codex supports deep integration with popular communication platforms through the configure-notifications skill.

#Skill-Based Setup

Instead of manual configuration, OMX uses the configure-notifications skill to set up integrations interactively. Simply invoke the skill inside Codex CLI:

Invoke the Configuration Skill

/configure-notifications

or ask naturally:

"configure discord notifications"

The skill will guide you through:

  • Selecting which platform to configure (Discord, Telegram, Slack, or OpenClaw)
  • Choosing webhook or bot integration
  • Securely storing credentials in your environment
  • Testing the connection

#Discord Integration

Discord is the most popular way to receive real-time alerts from OMX.

Setup Options

  • Webhook: One-way notifications to a channel
  • Bot API: Two-way communication (reply to control OMX)

Just say: "setup discord webhook" or "configure discord bot"

#Telegram Bot

Telegram offers excellent support for interactive bots and secure communication.

Configuration

Simply invoke:

"configure telegram notifications"

The skill will guide you through creating a bot with @BotFather and getting your Chat ID.

#OpenClaw Gateway

OpenClaw is a programmable gateway for deep agentic integration.

Custom Integration

For OpenClaw or custom webhook endpoints:

"configure openclaw gateway"

The skill will prompt for your HTTP endpoint URL and any required headers.

#Notification Routing

Control exactly what notifications you receive and how they are formatted.

Verbosity Level Description
minimal Only Start and End events. Zero noise.
session Start, Stop, Idle, and Input Required. Includes a small tmux tail. (Default)
agent Every agent turn is logged with its summary.
verbose Full tool-call stream and agent thoughts. Extreme detail.

Ready to configure?

Invoke the skill to set up all your integrations interactively.

/configure-notifications
