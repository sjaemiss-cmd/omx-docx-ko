#Magic Keywords

OMX detects your intent using "Magic Keywords". These trigger specialized modes without requiring the $ prefix.

ralph

Triggers the persistence loop. Codex will keep trying until the architect verifies the change. Automatically includes ultrawork.

"ralph fix all the bugs"

ralplan

Triggers iterative consensus planning. Planner, Architect, and Critic agents deliberate before execution.

"ralplan this new feature"

ulw / ultrawork

Triggers max parallelism. Splits tasks across multiple workers for maximum throughput.

"ulw refactor the API"

plan

Triggers the strategic planning interview to clarify requirements before starting.

"plan the new endpoints"

Stopping Execution

You can stop any active mode by simply saying:

COMMANDS
"stop"
"cancel"
"abort"

#Workflow Examples

Combine skills to create powerful, automated engineering pipelines.

The "Full-Auto" Pipeline

Best for large features. Combines consensus planning, parallel execution, and persistent verification.

ralplan
→
team
→
ralph
BASH
# 1. Plan with consensus
$ralplan "implement user dashboard"

# 2. Build in parallel
$team 4 "follow the plan in .omx/plans/latest.md"

# 3. Final polish and verification
$ralph "ensure dashboard is 100% complete"

The "Bug-Crusher" Loop

Best for fixing complex bugs that require multiple attempts and testing cycles.

plan
→
ralph
→
ultraqa
BASH
# 1. Investigate and strategize
$plan "fix the race condition in auth.js"

# 2. Implement fix with persistence
$ralph "apply the fix and verify"

# 3. Exhaustive QA cycle
$ultraqa "run all edge cases"

#Usage Examples

Invoking Skills

Skills can be invoked using the $ prefix or via the /skills magic command.

OMX
$analyze "why is the build slow?"
/skills:analyze "optimize the build"

Agent Role Prompts

Load specialized agent personas using /prompts:name.

OMX
/prompts:architect "review this system design"
/prompts:test-engineer "write unit tests for this module"
